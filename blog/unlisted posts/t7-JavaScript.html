A comparison of Ruby hashes and Javascript Objects

May 25th, 2015

Summary:

Javascript Objects and Ruby Hashes behave very similarly, though they differ in a number of important ways.

You hand me at {}'s

Both Ruby hashes and JS objects are signified with a pair of curly brackets, which gives us our first hint of thier shared functionality. Looking at a JS object:

  
    var object = {};
    --> {}

and a ruby hash:

  
    object = {}
    --> {}

it is pretty clear that they atleast look the same. Truely, they do share some functionality beyond appearance. You can add/store/retrieve key/value pairs from either structure, and itterate over their values.You can make lists of their keys, or values.

So what's the difference?

One of the major differences is that Javascript objects can behave more like class instances in Ruby, because objects can store functions. Ruby hashes can't store methods, because a method isn't an object. For example, this will work in javascript, but won't in Ruby:

  
    var object = {first: 1, second: 2, this.sum = function(){this.first + this.second}};
    object.sum();
    --> 3
