Relatively Fixed or Absolutely Confused?

The CSS position property, and all the values we can find

Sunday April 18th, 2015

Summary:

Being able to place elements on a web page is one of the most important aspects of web-design, but which choosing positioning value to use can be frustrating. The CSS position property is an opaque and obscure concept because the values are intuitive in many ways. Exploring the position property, however, can give a developer some clarity.

The Positioning property explained

Like almost all CSS properties, the position property does not affect the functioning of your HTML webpage. It affects the layout of the webpage. The layout is how we talk about where everything is in relation to everything else.

To be clearer, the position property in CSS tells your browser where to put something in relation to something else. That is it. It is like if you were standing in a room, and I wanted to get you to move towards the door. I can tell you, "From where you are now, go forward three steps", or I could tell you, "Go to the north wall, three paces from the east wall". No matter what I tell you to do, it is always in relation to somewhere. The position property works the same way, it tells the browser to take an element and put it somewhere relative to something.

img {postition:relative; top:-20px;)

In the above example, the browser is being told, take the image and move it from where you think it should be twenty pixels up.

Fixed

The :fixed value for the position property is by far the easiest to get. It says, element-x is placed relative to the browser window, literally the rectangular outline of what we can see. Like the rest of the positioning elements, you declare how far away the element should be from the top, bottom, left or right of the screen. Elements that are fixed do not scroll, and are taken out of the normal flow of the document.

Static is actually the easiest

Position:static; is actually the easiest to understand, because it is how the html is written. First things first, one right below the other. We just don't think of the static very often since it is the browser default.

Relative

The value relative is a bit more obtuse. It says to place the element up, down, left, right of where it otherwise would have been rendered. This means if content is added above it in the html document, and therefor where is should be changes, it will change along with it. This is useful when the goal is to layer elements on top of eachother so that they are overlapping. Of note is the fact that postition relative does not remove the element from the normal flow of the document and will leave the space it would have taken empty.

Absolute

The absolute value for position is by far the most obtuse and unintuitive. An absolutely positioned element is moved in relation to its first positioned ancestor. Lets unpack that sentence a bit.

First, it only cares about ancestors, meaning elements that it is inside of. Put another way, it only cares about elements that "wrap" around it, that the parents start tag is before it and its end tag is after it. Second, the parent, grandparent, great... must be positioned. It must have a css declaration of position:something (note: if the parent is position:static, it won't be positioned). Finally, it looks untill it finds one, and then stops looking. Odd behavoir sometimes occurs because the element has a parent that is positioned without the developer realizing it.
